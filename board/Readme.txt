srboard는
mysql을 사용하지 않는 php게시판입니다.
아마 mysql을 사용하는 게시판보다
출력속도가 약간 빠를겁니다.
euc-kr버전과
UTF-8버전으로 배포하고 있습니다.
게시판예제는 http://sariputra.hosting.paran.com/srboard/
 
게시판은 다수를 생성할수 있고,
따로 정해진 생성할수 있는 게시판 수의 한계는 없습니다.
 
정말로 그 상태에서 테스트해본건 아니지만,
구조적으로 정해진 한계는 다음과 같습니다.
생성된 각 게시판은 최대99만 9999개의 게시물을 저장할수 있습니다.
각 게시판의 덧글은 최대999만 9999개까지 저장할수 있습니다.
각 게시판의 분류는 최대 99개까지 생성할수 있습니다.
가입회원은 최대 9만9999명까지입니다.

글쓰기는 위지윅으로 되어있고,
게시판의 스킨은 현재 14개. 만든이가 배포하고 있습니다.
 
목록형태
게시판의 목록형태를 관리자가 정할수도 있고, 사용자가 바꿔볼수도 있습니다.
목록형태는 다음과 같습니다.
제목형 :: 일반 게시판과 같은 모양
본문형 :: 본문이 나열되는 모양
요약형 :: 썸네일이미지가 있을경우 이미지와 본문일부가 출력되는 모양
갤러리형 :: 썸네일위주로 나오는 모양
-------------- 위 네가지는 외관을 기준으로하는 구분이고,그 다음은 기능적 구분입니다.
방명록형 :: 방명록 모양이고, 게시물제목이 없습니다.
rss리더형 :: 제목형과 같은 모양이고, rss리더 게시판을 만듭니다.
블로그형 :: 페이지 출력할땐 본문형, 검색일땐 제목형으로 출력합니다.
토론형 :: 활성화된 순으로 목록을 출력합니다. 최근덧글이 달린게 맨 위로 가는 방식.
----- 이상이 목록형태입니다. 위의 네가지는 목록상단우측의 버튼으로 사용자가 모양을 바꿔볼수 있습니다.

덧글
갤러리형을 제외한 모든 목록형태에서
덧글을 볼수도/쓸수도 있습니다. 그래서 덧글은 iframe으로 출력됩니다.
덧글에 덧글기능이 있습니다.
덧글에 덧글에 덧글은 안됩니다.
 
회원로그
회원이 쓴 글, 쓴 덧글은 전부 따로 수집되어서
회원로그에 출력됩니다.
쓴글은 rss피드로 출력할수 있습니다.(제목과 링크만 있고,본문은 없는)
게시판의 글을 스크랩(본문빼고, 목록만) 할수 있습니다.
회원본인만 볼수있는 다이어리가 제공됩니다.
 :: 관리자도 다른 회원의 다이어리를 볼수는 없지만,
저장된 데이타파일을 열어 볼수는 있습니다.
 
섹션
섹션은 게시판 머리부분의 메뉴의 제목을 가리킵니다.
게시판 여러개를 한꺼번에 묶어서 대문(최근게시물)으로 출력합니다.
섹션을 출력하는 섹션메뉴는 전체적인 레이아웃입니다.
현재 너댓가지를 만든이가 배포하고 있습니다.
게시판 좌우의 메뉴를 좌우메뉴라고 지칭하고 있습니다.
각 좌우메뉴는 이동/배치/순서가 자유롭습니다.
좌우메뉴는 현재 20여가지가 있습니다.
로그인부터 기가라이브채팅까지..

경로이동 ::
왼쪽 경로로 이동합니다
파일생성, 폴더생성 ::
왼쪽에 생성할 파일/폴더의 경로를 넣고 버튼을 누르면 생성됩니다.
물론 경로가 PHP가 파일을 생성할수 있는 권한이 있는 경로여야 합니다.
PHP에서 생성된 폴더이거나, FTP로 생성되었을때는 권한이 707, 777 이어야 합니다.
폴더는 경로이동 /파일은 다운로드 ::
아래 폴더/파일목록에서 파란색의 폴더이름 클릭하면 경로이동
파일이름 클릭하면 다운로드 됩니다.
파일크기 ::
소유자 ::
대개 99로 되어 있는데, PHP에서 생성된 폴더는 소유자가 nobody 99로 표시됩니다.
편집 ::

누르면 위와 같은 파일 내용편집창이 뜹니다.
이미지 넓이는 조정했습니다.

경로이름이 있는데, 여기서 이름을 바꿔서 저장하면 내용이 그 새이름의 파일로 생성됩니다.
검색교체는 자바스크립트 정규식 검색입니다.
따라서 검색어에는 정규식 특수문자 (){}[].?+*|^$/ 를 유의해야 합니다.
저 특수문자를 검색할 때에는 앞에 \ 백슬래시를 넣어야 합니다.
검색에서 치환자를 사용할수도 있습니다.

파일크기가 1mb 이상인경우엔,
내용을 전부 textarea에 출력하는게 브라우저에 부담이 되기 때문에 (에러-먹통)
파일크기가 1mb 이하인 경우에만 내용편집 가능하도록 했습니다.

1mb 이상인 파일에서 내용편집이 필요한 경우엔
앞에 체크하고 15번 0777을 누르고.
FTP기능이 있는 에디터로 편집저장하면 됩니다.
검색 ::

검색은 검색교체의 준말(?)입니다.
파일에서 검색해서 교체합니다.
정규식 검색이고, 치환자를 사용할수 있습니다.

내용편집과는 달리 파일이 커도 실행됩니다.
그리고 내용편집의 자바스크립트 정규식검색보다 처리시간이 빠릅니다.

내용편집의 검색과 비슷하게
정규식 특수문자 (){}[].?+*|^$`는 유의해야 합니다.
내용편집의 정규식특수문자와 끝에 하나가 다릅니다.
내용편집의 끝에 하나는 / (슬래시)
검색교체의 끝은 ` (shift ~)

예를 들어 검색어에 @([a-z]+)를 넣고 교체어에 (at)$1 를 넣으면,
xxx@naver.com 같은 구문을 검색해서 xxx(at)naver.com 으로 교체시킵니다.
@ 다음에 [a-z]+를 검색하고, 괄호로 싸서 기억해두었다가 $1로 되살린 것입니다.
수정일 / 링크 ::
파일/폴더의 수정일입니다. 그리고 텍스트에는 그 파일/폴더로의 링크가 걸려 있습니다.
다만 23번의 .htaccess 파일에서 srboard의 data폴더 내부로의 외부접근은 전부 야후로 돌려버리기 때문에
스샷의 경로에서는 링크가 안됩니다.
폴더삭제 ::
확인창 한번 더 뜨고, 거기서 확인을 선택하면, 폴더가 삭제됩니다. 그 내부의 파일도 모두 삭제됩니다.
단 PHP가 해당 폴더와 그 내부의 파일을 삭제할 권한이 있을 경우에
삭제 / 비움 ::
삭제는 파일을 삭제하는 것이고, 비움은 파일내용을 비워서 이름만 남은, 파일크기 0의 파일로 만듭니다.
폴더크기 ::
보여지는 값은 폴더 내부의 모든 파일 크기를 합산한 크기입니다.
폴더가 상위로 갈수록, 파일크기 합산하는 과정이 길어지기 때문에 출력이 늦어집니다.
전체설정에 파일관리에 폴더 내부크기 출력여부를 선택할수 있도록 해서,
폴더내부 크기가 궁금하지 않은 경우엔, 그 합산과정을 생략할수 있게 했습니다.
선택 ::
버튼을 누르면 체크박스 선택반전이 됩니다.
copy ::

복사할 파일/폴더(들)을 체크하고,
copy 버튼을 누르면, 왼쪽에 복사할 경로란이 나옵니다.
파일 하나만 복사할 경우는 경로란에
복사해서 생성할 파일경로와 이름을 적어주면 되고,
그 외의 경우엔,
복사해 넣을 폴더의 경로를 넣어주면 됩니다.
경로를 적어주고 우측에 긴 copy 버튼을 클릭하면 됩니다.

폴더도 복사가 됩니다.
rename ::

rename도 copy처럼 경로란이 나옵니다.
사용법도 copy하고 같습니다.
파일/폴더 여러개를 선택했을때는, 경로에 선택한 파일을 이동시킬 폴더의 경로를 적어주면 되고,
파일 하나만 선택했을때는, 선택파일을 이동시킬 폴더의 경로를 적어주거나,
경로와 함께, 새로운 파일이름을 적어주면 됩니다.

파일/폴더 하나만 선택해서 새로운 이름으로 바꿀때는 이름변경이고,
여러개 선택했을때는, 이름변경보다는 파일이동이라고 생각하는게 좋겠습니다.

copy와 rename이 다른 점은 두가지
첫째, copy는 복사되는 파일은 그냥 두고, 또 다른 파일이 생성된다는 점
둘째, copy는 복사생성되는 파일/폴더의 소유자가 99로 바뀌는데, rename은 바뀌지 않는다는 점.

rename에서는 파일소유자가 안바뀌기 때문에,
srboard의 데이타파일을 백업했다가, 다시 환원시킬때
백업했던 데이타파일을 FTP로 계정에 업로드한 경우
FTP에 로그인한 사람으로 파일소유자가 기록되는거 같습니다.
그래서 FTP에서 업로드한 백업파일 그 자체로는
srboard (PHP)가 읽고 쓰고 할수가 없어서,
PHP가 읽고 쓰고 할수 있도록 바꿔줘야 하는데,
copy로는 그게 되는데, rename으로는 안됩니다.

더 자세한 얘기는
http://srboard.styx.kr/srboard/index.php?id=srboard&no=5
링크를 참고하시고, 하여튼 백업한 데이타파일 되살릴적에 
여기 파일관리에 copy를 쓴다는 정도만 알아도 될거 같습니다.
0777 ::
srboard에서 생성된 데이타파일을 FTP기능이 있는 에디터로 열어볼수는 있는데,
편집하고 저장하려고 하면, 권한이 없다고 하면서 저장이 안됩니다.
그런 경우에 그 파일에 체크하고 0777버튼을 누르면
FTP기능이 있는 에디터에서 편집저장할수 있게 됩니다. (일시적으로)
압축 ::
압축기능은 PHP함수자체에는 없기 때문에
쉘스크립트를 씁니다. (영체제 리눅스의 명령어를 PHP에서 실행시키는 것)
그런데 그게 막혀있는 계정도 있습니다.
그런 경우엔 서버의 설정을 검색해서,압축과 압축해제 버튼을 출력하지 않습니다.

압축할 파일/폴더(들)을 체크하고, 압축버튼을 누르면
data/암호폴더/날짜.tar.gz 위치에 압축파일이 생성됩니다.
압축해제 ::
압축파일의 압축을 풉니다.
검색 ::

먼저 검색이 이뤄질 파일/폴더(들)을 체크합니다.
버튼을 누르면 검색어를 넣을수 있는 란이 나옵니다.
copy, rename에서의 경로란을 같이 쓰다보니까 앞에 경로 : 라고 이름이 붙어있는데,
하여튼 검색어를 넣고 find 버튼을 누르면 검색된 파일목록이 목록에 뜹니다.
검색은 정규식 검색이 아니고, 그냥 일반적인 검색입니다.
삭제 ::
체크된 파일/폴더를 삭제합니다. 폴더의 경우 그 내부의 모든 파일을 포함해서 삭제합니다.
누르면 확인창이 뜹니다.
비움 ::
체크된 리스트에서 파일의 경우에만 적용됩니다. 폴더가 체크되어 있어도 에러는 안납니다.
파일내용을 비우고, 이름만 남은 파일로 만듭니다.
폴더가 체크되었더라도, 내부파일 추적은 안합니다. 그냥 건너뜁니다.
파일업로드 ::
파일을 업로드합니다. 업로드되는 곳은 현재 보여지는 경로입니다.
PHP에서 파일생성할수 있는 권한이 있어야 업로드됩니다.
암호폴더 ::
srboard의 데이타파일을 외부접근으로부터 보호하기 위해서,
srboard를 설치(?)할때, 시간정보와 이것저것으로 암호폴더를 생성하고,
그 안에 데이타파일을 저장합니다.
암호폴더의 이름은 아무곳에도 저장되어있지 않습니다.
srboard의 게시판소스가 접근할때는,
data폴더를 읽어서, 그 안의 내역으로 경로를 파악합니다.
따라서,,,, 암호폴더의 이름은 언제나, 아무렇게나 바뀌어도 됩니다.
그리고 data폴더 안에 이 암호폴더외에 다른 파일이 있으면 안됩니다.
(게시판소스가 데이타파일 경로를 오인할수 있으니까)

srchat 채팅소스에서 채팅방 데이타파일보호를 위해서, 했던 기능인데,
srboard에 적용시킨 것입니다.
.htaccess ::
내용은 아래와 같습니다.
RewriteEngine On
RedirectMatch /(.*)$ http://www.yahoo.com
그냥... 모든 외부접근을 야후로 돌려버리는 것입니다.
아파치서버에서만 실행됩니다.

이것과 위에 암호폴더 두가지로 데이타파일을 보호하고 있습니다.
게시판의 업로드파일은
data/암호폴더/게시판아이디/files/ 경로에 저장되는데,
암호폴더 안쪽으로 외부접근이 안되기 때문에,
파일의 다운로드는 PHP에서 읽어서 뿌려주는 방식으로 되어 있습니다.
http://게시판주소/exe.php?id=게시판아이디&file=파일이름
http://게시판주소/exe.php?id=게시판아이디&fno=파일번호
이 두가지 방식으로 다운로드/파일출력이 이뤄지는데,
두번째 방식은 다운로드 횟수를 카운트하는 것입니다.

회원정보
글쓴이 : 사리   2009-03-17 22:02    

쓴글,덧글,출석 회수는 관리자기능-전체설정의 가중치를 곱해서, 포인트에 더해집니다.
포인트 ::

포인트 옆에 숫자버튼이 있는데, 숫자는 포인트 값이고, 
버튼을 누르면 위와 같은 포인트 내역창이 뜹니다.

관리자만 회원의 포인트를 높이거나 낮출수 있습니다.
위 이미지는 관리자 상태의 이미지라서 new란과 추가버튼이 있는데,
일반회원의 포인트내역창에는 그 두항목이 없습니다.
포인트를 높일때는 new 포인트란에 정수값을,
포인트를 낮출때는 -정수 값을 주면 됩니다.
받은 쪽지함 ::

받은 쪽지함 스샷입니다.
받은 쪽지는 한 페이지에 열개씩 나열됩니다.
빨간색의 X를 클릭하면, 쪽지가 나열되지 않습니다.
보낸이가 회원이 아니라면, 받은이가 쪽지를 삭제할때, 기록자체가 삭제되지만,
보낸이가 회원이라면, 받은이와 보낸이가 모두 삭제해야 비로소 기록이 삭제됩니다.
(하나의 기록을 가지고, 보낸이와 받은이 모두에게 출력하고 있기 때문에)
회원로그 ::
회원이 쓴글, 쓴덧글, 스크랩한 글, 출석부, 다이어리 등을 출력하는 것입니다.
프로필이미지 ::
게시판목록등에서 보여지는 이미지입니다.
icon/m12_회원번호 경로로 저장됩니다.
80x80 크기는 그냥 권장사항입니다.
서명문구 ::
본문 하단에 출력하는 서명문구입니다.
단 관리자기능-게시판관리에서 서명문구를 노풀하도록 설정되어 있어야 노출됩니다.
icon/m02_회원번호 경로로 저장됩니다.
비밀번호변경 ::
버튼을 누르면 새로운 비밀번호를 입력하는 란이 두개 나옵니다.
별표로 표시되니까, 오타방지를 위해서 새로운 비밀번호를 두번 입력하게 되어 있습니다.
비밀번호는 암호화 저장되어서
관리자가 저장된 비밀번호를 보고도, 뭔지 알수가 없습니다.
그래서 회원이 비밀번호를 잊어버렸을때는, 
관리자에게 연락해서 비밀번호를 재설정해달라고 해야 합니다.

관리자기능 회원관리에 있는 비밀번호 재설정(비번변경)으로 새로운 비밀번호를 입력하고,
그 새로운 비밀번호를 회원에게 알려서, 로그인할수 있게 하고,
그 재설정된 비밀번호는 관리자가 알고 있으니까,
로그인후에 여기서 비밀번호를 다시 변경하면 됩니다.
홈페이지 ::

값이 있으면, 닉네임 팝업창에 홈페이지링크가 뜹니다.
닉네임 ::
닉네임을 수정할때는 우측에 [중복확인]을 거쳐야 합니다.
닉네임을 수정하면 우측에 [ok]가 [중복확인]으로 바뀝니다.
이메일변경 ::
이메일주소를 바꿀때는 유효성검사를 하게 됩니다.
이메일주소에 사이트가 있는건지 정도를 검사합니다.
성별선택 ::
관리자기능 전체설정에서 성별선택 강제여부를 지정하도록 되어 있습니다.
강제함으로 지정된 경우엔, 반드시 성별을 지정해야 합니다.
다이어리 공개설정 ::
회원로그에 다이어리를 공개할 것인지, 누구에게 공개할 것인지를 선택합니다.

관리자의 경우는 회원정보 링크를 클릭하면, 관리자기능으로 넘어가고,
관리자기능에서 관리자정보를 클릭해야 회원정보창이 뜹니다.
일반회원의 경우는, 회원정보를 클릭하면 바로 회원정보창으로 갑니다.
